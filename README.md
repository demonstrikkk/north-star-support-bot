# North Star Support Bot

A complete, evaluator-ready customer-support chatbot for a simulated outdoor apparel and camping-gear business. The experience combines a responsive galaxy/aurora interface with deterministic, typo-tolerant conversation logic.

## Required use cases

- Order tracking for orders `111`, `222`, and `333`
- Invalid-order handling
- Returns and exchanges policy
- Product-category recommendations using two guided questions
- Shipping information
- Progressive fallback handling
- Simulated Live Agent handoff
- Return to the main menu after every flow

## Technology

- **Frontend:** React, TypeScript, Vite, Tailwind CSS, Framer Motion, Lucide React
- **Backend:** FastAPI, Pydantic, RapidFuzz
- **Tests:** Pytest, Vitest, React Testing Library

## No-API-key guarantee

> The required evaluation mode uses no external AI model, API key, paid service, or third-party account. All business responses are generated from the supplied project data and deterministic conversation logic.

## Quick start — production build included

```bash
python -m venv .venv
# Windows: .venv\Scripts\activate
# macOS/Linux: source .venv/bin/activate
pip install -r requirements.txt
python run.py
```

Open `http://localhost:8000`.

The ZIP includes the compiled React application under `app/static`, so Node.js is not required for ordinary evaluation.

## Frontend development

```bash
cd frontend
npm install
npm run dev
```

In a second terminal:

```bash
pip install -r requirements.txt
uvicorn app.main:app --reload
```

## Exact supplied business data

| Order | Required status |
|---|---|
| `111` | Shipped, arriving tomorrow |
| `222` | Processing, ships in 24 hours |
| `333` | Delivered |
| Any other order | Invalid order |

Return policy:

- Returns within 30 days
- Items must be unused
- Original packaging is required

Shipping:

- Standard: 3–5 business days
- Expedited: 1–2 business days

## Intent handling

The backend uses:

1. Global command detection
2. Active-flow state resolution
3. Order-number extraction
4. Curated phrase matching
5. Weighted keywords
6. RapidFuzz typo tolerance
7. Progressive fallback and handoff

It supports natural messages such as:

- `Track order 111`
- `Where is my parcel?`
- `trak my pakage`
- `I received my order and want to return it`
- `Help me choose camping gear`
- `speek to a human`

## Docker deployment

```bash
# Build and start
docker compose up --build -d

# Stop
docker compose down

# View logs
docker compose logs -f
```

The container runs on port `8000` by default. Set environment variables in `docker-compose.yml` or a `.env` file:

| Variable | Default | Description |
|---|---|---|
| `HOST` | `0.0.0.0` | Bind address |
| `PORT` | `8000` | Listen port |
| `UVICORN_WORKERS` | `1` | Number of worker processes |
| `LOG_LEVEL` | `info` | Log level (debug, info, warning, error) |
| `CORS_ORIGINS` | `*` | Comma-separated allowed origins or `*` |

## Key design decisions

- Business facts are deterministic and never generated by an open-ended model.
- Product recommendations return categories only because no product catalogue was supplied.
- The Live Agent flow is explicitly simulated.
- The local returns route is demonstrative and never claims to submit a real return.
- FastAPI serves the production React build so evaluators need only one running process.

## Limitations

- Session state is in memory and resets when the server restarts.
- No real order database, carrier API, ecommerce platform, CRM, or human support system is connected.
- Only supplied business information is answered.
